# ShortFlow 커맨드 사용 매뉴얼

> 마지막 업데이트: 2026-04-09

---

## 개요

ShortFlow는 숏핑(쇼핑 쇼츠) 유튜브 영상 제작을 자동화하는 AI 워크플로우입니다.
Claude Code에서 슬래시 커맨드로 실행하며, 두 가지 핵심 커맨드로 구성됩니다.

```
/research-product  →  제품 리서치 및 01_products.md 생성
/make-video        →  레퍼런스 분석부터 업로드 정보까지 영상 제작 전 과정
```

**권장 실행 순서:**

```
/research-product  →  /make-video
```

---

## 커맨드 1: `/research-product`

### 한 줄 설명

영상에 쓸 제품을 확정하고 리서치 파일(`01_products.md`)을 생성합니다.

### 언제 사용하나요?

- 새 영상을 시작할 때 **가장 먼저** 실행
- 제품이 이미 정해진 경우에도, 추가 리서치 및 함께 소개할 제품 탐색에 활용

### 실행 방법

Claude Code 채팅창에 입력:

```
/research-product
```

### 인터랙션 흐름

```
Claude: "이미 영상으로 만들 제품이 정해졌나요?"
         1. 네, 제품이 있어요
         2. 아니요, 트렌드 제품을 찾아주세요
              ↓
[선택 1: 제품 있음]              [선택 2: 제품 없음]
제품명 → 링크 → 기타 정보         구독 채널 6개 순서대로 분석
추가 웹 리서치 자동 수행           트렌드 제품 목록 제시
              ↓                          ↓
     "함께 소개할 제품도 필요한가요?"
     단독(B형) vs 묶음(A형) 선택
              ↓
     묶음 선택 시: 구독 채널에서 어울리는 제품 자동 탐색
              ↓
         01_products.md 저장
```

### 구독 채널 탐색 순서

| 우선순위 | 채널명 | 비고 |
|----------|--------|------|
| 1 | 살림토끼 | 숏핑 적합도 높음 |
| 2 | 살림남 | 숏핑 적합도 높음 |
| 3 | 모닝살림 | 숏핑 적합도 높음 |
| 4 | 오!득템 | 숏핑 적합도 높음 |
| 5 | 쇼핑맘 살구홈 | 빠르게 확인 |
| 6 | 꿀팁스쿨 | 빠르게 확인 |

### 출력물

```
workspace/
└── {YYYY-MM-DD}_{제품명약칭}/
    └── 01_products.md   ← 제품 리서치 결과
```

`01_products.md` 포함 내용:
- 제품명, 가격, 구매처 링크
- 주요 기능 및 특징
- 실사용 후기 키워드
- 레퍼런스 영상 URL
- 영상 유형 (A형/B형) 및 테마 방향

## 커맨드 2: `/make-video`

### 한 줄 설명

레퍼런스 분석 → 대본 → 소스 가이드 → 편집 가이드 → 업로드 정보까지,
영상 제작에 필요한 모든 파일을 단계별로 생성합니다. **각 단계마다 사용자 확인을 받습니다.**

### 실행 조건

`workspace/` 내에 `01_products.md`가 있어야 합니다. 없으면 먼저 `/research-product`를 실행하세요.

### 실행 방법

**Step 1부터 전체 실행:**

```
/make-video
```

**특정 단계부터 시작 (이미 일부 진행한 경우):**

```
/make-video 2   → Step 2(대본 작성)부터
/make-video 3   → Step 3(소스 수집 가이드)부터
/make-video 4   → Step 4(편집 가이드)부터
/make-video 5   → Step 5(업로드 정보)부터
```

### 단계별 상세 내용

#### Step 1 — 레퍼런스 분석 → `02_reference.md`

| 항목 | 내용 |
|------|------|
| 하는 일 | 유사 제품 고조회수 쇼츠 3개 이상 분석 |
| 분석 내용 | 오프닝 후킹 문장, 컷 구성, 자막 스타일, 영상 소스 유형 |
| 확인 질문 | "이 패턴으로 대본을 작성할까요?" |

#### Step 2 — 대본 작성 → `03_script.md`

| 항목 | 내용 |
|------|------|
| 하는 일 | 레퍼런스 패턴 기반 숏핑 대본 작성 |
| 포함 내용 | 오프닝 후킹 3안, 컷별 자막·영상컷·시간 테이블 |
| 영상 유형 | A형(3종 묶음, 22~28초) / B형(1종 집중, 40~60초) |
| 확인 질문 | "수정할 부분이 있으신가요?" |

> 이 단계에서 수정 요청 가능. "3번째 컷 바꿔줘" 등 구체적으로 요청하면 반영 후 재확인.

#### Step 3 — 소스 수집 가이드 → `04_sources.md`

| 항목 | 내용 |
|------|------|
| 하는 일 | 컷별 이미지/영상 소스 수집 방법 안내 |
| 소스 유형 | AI 이미지(Google Whisk), AI 영상(Runway), 쿠팡 캡처, 직접 촬영 |
| 포함 내용 | Whisk/Runway 영문 프롬프트, 수집 체크리스트 |
| 확인 질문 | "계속 진행할까요?" |

#### Step 4 — CapCut 편집 가이드 → `05_edit_guide.md`

| 항목 | 내용 |
|------|------|
| 하는 일 | CapCut 타임라인 가이드 및 체크리스트 생성 |
| 포함 내용 | 컷별 시작/끝 시간, 소스 파일명, 적용 효과, 자막 타이밍 |
| 설정 기준 | 9:16, 1080x1920, 30fps, 자막 흰색+검정 테두리 |
| 확인 질문 | "업로드 정보도 만들까요?" |

#### Step 5 — 업로드 정보 생성 → `업로드_정보.md`

| 항목 | 내용 |
|------|------|
| 하는 일 | YouTube 업로드에 필요한 모든 텍스트 준비 |
| 포함 내용 | 제목 후보 3개, 설명란, 해시태그 15개, 썸네일 문구 3줄 |
| 추가 안내 | 쿠팡 파트너스 링크 등록 방법 |

### 전체 출력물 구조

```
workspace/
└── {YYYY-MM-DD}_{제품명약칭}/
    ├── 01_products.md      ← /research-product 생성
    ├── 02_reference.md     ← Step 1
    ├── 03_script.md        ← Step 2  ★ 핵심 파일
    ├── 04_sources.md       ← Step 3
    ├── 05_edit_guide.md    ← Step 4
    └── 업로드_정보.md       ← Step 5
```

### 사용자 확인 포인트

`/make-video`는 각 단계 완료 후 자동으로 멈추고 사용자 확인을 기다립니다.
수정이 없으면 "다음 단계" 또는 "계속"이라고 입력하면 됩니다.

## 완료 후 수동 작업

`/make-video`까지 완료되면 아래 순서대로 직접 진행합니다.

```
1. 🎙  TTS 제작      Typecast (https://typecast.ai)
                     03_script.md의 자막 컬럼 텍스트 입력

2. 🖼  AI 이미지 생성 Google Whisk (https://labs.google/fx/tools/whisk)
                     04_sources.md의 프롬프트 사용

3. 🎬  영상 편집      CapCut 데스크톱 앱
                     05_edit_guide.md 타임라인 가이드 따라 편집

4. 🛒  파트너스 링크  쿠팡 파트너스 (https://partners.coupang.com)
                     링크 생성 → 업로드_정보.md 설명란에 삽입

5. 📱  YouTube 업로드 YouTube Studio
                     업로드_정보.md 내용 복사 붙여넣기
```

## 자주 쓰는 시나리오

### 시나리오 A: 제품이 없을 때 (풀 자동)

```
/research-product
→ "아니요, 트렌드 제품을 찾아주세요" 선택
→ 채널 분석 결과에서 제품 선택
→ /make-video
→ 각 단계 확인하며 진행
```

### 시나리오 B: 제품이 있을 때 (단독 영상)

```
/research-product
→ "네, 제품이 있어요" 선택
→ 제품명/링크/정보 입력
→ "단독 집중(B형)" 선택
→ /make-video
```

### 시나리오 C: 제품이 있을 때 (묶음 영상)

```
/research-product
→ "네, 제품이 있어요" 선택
→ 제품명/링크/정보 입력
→ "묶음 소개(A형)" 선택
→ 함께 소개할 제품 채널 탐색 후 선택
→ /make-video
```

### 시나리오 D: 대본만 다시 만들고 싶을 때

```
/make-video 2
→ 기존 01_products.md + 02_reference.md 읽고 대본 재작성
```
