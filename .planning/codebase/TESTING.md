## Validation Approach

### 1. 산출물 형식 검증 (수동)

각 스킬의 출력 MD 파일이 SKILL.md에 정의된 템플릿과 일치하는지 확인:

| 산출물 | 필수 섹션 | 검증 포인트 |
|--------|-----------|-------------|
| `01_products.md` | 채널별 인기 제품, 제작 추천 | S/A/B 등급 존재, 3순위 이상 |
| `02_reference.md` | 영상 목록, 상세 분석, 성공 패턴 | 최소 3개 영상, 후킹 패턴 포함 |
| `03_script.md` | 기획서, 후킹 후보, 대본 테이블 | 22~60초 범위, 테이블 형식 |
| `04_sources.md` | 컷별 소스, AI 프롬프트 | 모든 컷 커버, 영문 프롬프트 |
| `05_edit_guide.md` | 타임라인, 자막 설정, 체크리스트 | 초 단위 타이밍, 9:16 설정 |
| `업로드_정보.md` | 제목 후보, 설명란, 해시태그 | 3개 제목, 15개 해시태그 |

### 2. 워크플로우 연속성 검증

에이전트 체인이 올바르게 연결되는지 확인:
- 각 에이전트가 올바른 입력 파일을 읽는지
- 출력 파일이 다음 에이전트의 입력으로 사용 가능한지
- 사용자 개입 시점(Step 1, 3, 6)에서 정확히 멈추는지

### 3. End-to-End 검증

전체 워크플로우 실행 후:
- [ ] 6개 산출물 파일 모두 생성됨
- [ ] 대본 길이가 목표 범위 내 (22~60초)
- [ ] AI 이미지 프롬프트가 영문으로 작성됨
- [ ] 쿠팡 파트너스 링크 삽입 안내 포함
- [ ] CTA "제품정보는 프로필 링크에!" 포함

## Quality Gates

현재 자동화된 품질 게이트 없음. 사용자 검토가 유일한 품질 관리:

1. **제품 선택** (Step 1 후): 제작할 제품 적절성 판단
2. **대본 검토** (Step 3 후): 후킹력, 길이, 정보 정확성
3. **최종 확인** (Step 6 후): 전체 패키지 완성도
