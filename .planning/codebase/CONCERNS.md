## High Priority

### 1. MD 파일 기반 컨텍스트 전달 취약성
- **위치**: 전체 에이전트 체인
- **문제**: 6개 에이전트가 MD 파일로만 정보 전달 — 형식이 깨지면 후속 에이전트가 파싱 실패
- **영향**: 전체 파이프라인 중단
- **제안**: MD 스키마 검증 + 구조화된 섹션 마커 도입

### 2. 에러 핸들링 전무
- **위치**: 모든 `SKILL.md` 파일
- **문제**: 정상 경로(happy path)만 정의 — 제품 미발견, 레퍼런스 영상 없음, 대본 길이 초과 등 실패 시나리오 미정의
- **영향**: 에이전트가 예외 상황에서 멈추거나 잘못된 결과 생성
- **제안**: 각 SKILL.md에 에러 상태 + 폴백 로직 섹션 추가

### 3. 외부 서비스 의존성 — 대안 경로 없음
- **위치**: `trend-researcher/SKILL.md`, `source-collector/SKILL.md`
- **문제**: YouTube, Typecast, Google Whisk, 쿠팡 파트너스에 하드 의존 — 서비스 중단/정책 변경 시 워크플로우 마비
- **영향**: 전체 워크플로우 불능
- **제안**: 각 외부 서비스별 폴백 옵션 정의

## Medium Priority

### 4. YouTube 스크래핑 방식의 불안정성
- **위치**: `trend-researcher/SKILL.md` (Step 1)
- **문제**: JavaScript DOM 셀렉터(`querySelectorAll('a[href*="/shorts/"]')`)로 영상 추출 — YouTube 페이지 구조 변경 시 즉시 중단
- **영향**: Step 1 실패 → 전체 워크플로우 시작 불가
- **제안**: YouTube Data API v3 또는 셀렉터 버전 관리

### 5. 대본 길이 검증 자동화 없음
- **위치**: `script-writer/SKILL.md`
- **문제**: 대본 테이블의 시간 합산이 22~60초 범위인지 자동 검증하지 않음 — 60초 초과 가능
- **영향**: 쇼츠 길이 제한 위반, 편집 단계에서 재작업
- **제안**: 대본 작성 후 자동 시간 합산 검증 단계 추가

### 6. AI 이미지 프롬프트 품질 미확인
- **위치**: `source-collector/SKILL.md` (Step 2)
- **문제**: 생성된 영문 프롬프트의 실제 이미지 품질을 사전 확인하지 않음
- **영향**: 사용자가 Whisk에서 반복 시도 필요
- **제안**: 프롬프트 예시 + 스코링 가이드 추가

### 7. 워크스페이스 관리 부재
- **위치**: `shorts-pd/SKILL.md`
- **문제**: 워크스페이스 폴더 생성/정리/인덱싱 자동화 없음 — 과거 프로젝트 추적 어려움
- **영향**: 프로젝트 이력 관리 불가
- **제안**: shorts-pd에 Step 0(워크스페이스 초기화) + 인덱스 파일 추가

## Low Priority

### 8. CapCut 편집 100% 수동
- **위치**: `edit-guide/SKILL.md`
- **문제**: CapCut은 외부 API 없음 — 가장 시간 소요가 큰 단계가 완전 수동
- **영향**: 전체 워크플로우 병목
- **제안**: 프로젝트 템플릿 활용, 자막 자동 생성 스크립트 검토

### 9. 쿠팡 파트너스 링크 추적 부재
- **위치**: `upload-manager/SKILL.md`
- **문제**: 링크 수동 생성, 영상별 성과 추적 없음
- **영향**: 수익화 최적화 불가
- **제안**: 링크 데이터베이스(CSV/JSON) + 성과 추적 대시보드

### 10. 모니터링/로깅 없음
- **위치**: 전체 워크플로우
- **문제**: 에이전트 실행 로그, 비용 추적, 성능 메트릭 없음
- **영향**: 디버깅 어려움, 워크플로우 최적화 불가
- **제안**: workspace/LOG.md + 실행 요약 리포트 자동 생성

### 11. 설정 파일 우선순위 불명확
- **위치**: `.claude/settings.json`, `.claude/settings.local.json`
- **문제**: 두 설정 파일 간 병합/우선순위 규칙이 문서화되지 않음
- **영향**: 예기치 않은 권한 동작
- **제안**: CLAUDE.md에 설정 우선순위 명시

## Security Notes

- **API 키 없음**: Claude Max 플랜 내 운영으로 외부 API 키 불필요
- **민감 데이터 없음**: 쿠팡 파트너스 링크는 공개 URL
- **YouTube 접근**: DOM 스크래핑 — TOS 위반 가능성 있음 (API 전환 권장)
- **설정 파일**: `settings.local.json`에 도메인별 WebFetch 허용 목록 존재 — 의도된 범위 확인 필요
