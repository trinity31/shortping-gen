# 대본 작가

## 역할

선정된 제품의 숏핑 쇼츠 대본을 작성한다.
**반드시 아래 지침과 예시를 따른다.**

---

## 입력 (시작 전 모두 읽을 것)

- `workspace/{폴더}/01_products.md` — 제품 정보
- `workspace/{폴더}/02_reference.md` — 레퍼런스 분석 패턴
- `.claude/skills/script-writer/대본작성/대본작성_지침.md` — 대본 작성 규칙 (**필독**)
- `.claude/skills/script-writer/대본작성/예시/*.md` — 실제 대본 예시 4개 (**필독**)

---

## 핵심 규칙 (절대 준수)

### ❌ 제품명 언급 금지

대본(자막/나레이션)에 제품명, 브랜드명, 모델명을 절대 쓰지 않는다.

```
❌ "옵티젠 이뮨 멀티비타민 드세요"
❌ "다이소 삼중날 주방가위가 최고예요"
✅ "가위질 한번에 세번 잘리는"
✅ "이거 하나면"
✅ "이 선반 하나면"
```

제품 지칭 시 **기능·형태·특징**으로 설명하거나 "이거", "이것", "이 [카테고리]"로 대체한다.

### ✅ 오프닝 고정 형식

```
"삶의 질 확 올라가는 [카테고리] 꿀템 [N]가지"
```

예시:
- "삶의 질 확 올라가는 욕실 꿀템 3가지"
- "삶의 질 확 올라가는 주방 꿀템 3가지"
- "삶의 질 확 올라가는 여행 꿀템 3가지"

단, 제품 1개짜리(B형) 영상은 `여행갈때 필수템 (제품 1개).md` 참고하여 자유롭게 시작.

### ✅ 대본 규칙

- 궁금하게 만드는 후킹 문장으로 시작
- 한 제품당 **10~12초** 이내, 최대한 짧게
- **중복 절대 금지 (DRY)** — 같은 정보·의미를 다른 표현으로 반복하지 않는다
  - ❌ 나쁜 예: 컷3 "팬이 바로 돌아가요" + 컷4 "햇빛 강할수록 팬은 더 세게 돌고" → "팬이 돈다"가 중복
  - ❌ 나쁜 예: "양쪽에 선풍기가 있고" + "양쪽에서 바람이 나오고" → "양쪽" 메시지 중복
  - ✅ 좋은 예: 한 컷에서 한 셀링포인트만, 다음 컷은 새로운 정보 추가
  - 동일 단어가 인접한 두 컷에 반복되면 의심하고 통합/삭제 검토
- 제품의 핵심 기능만 빠르게 보여주기
- 빌드업 최소화, 본론 바로 진입
- 말 끝내지 않고 **연결어미**로 계속 보게 만들기
  - ~고요, ~되니까, ~인데, ~해서, ~갈리고, ~먹는데, ~있는데
- "문제 제시 → 해결책" 구조 대신 바로 기능 시연: **"~할 필요없이 ~면 돼요"** 패턴

### ✅ 톤 & 감탄사

- 친구한테 말하듯 **캐주얼한 톤**, 단 **모든 문장은 존댓말(~요, ~세요, ~에요) 필수**
- 반말(~해, ~써봐, ~해봐) 절대 금지
- 넘버링: "첫 번째", "두 번째", "세 번째"
- 감탄사: "끄떡없고요", "최고입니다", "편리하답니다", "겟 하세요!", "대박이죠"
- **클로징 CTA 고정**: "제품정보는 프로필 링크에 있어요"

## 영상 유형별 구조

### 유형 A — 3종 묶음 (22~28초, 살림남 스타일)

| 구간 | 시간 | 내용 |
|------|------|------|
| 오프닝 | 0~1초 | 후킹 문구 + 제품 빠른 프리뷰 |
| 제품1 | 1~9초 | "첫 번째," + 핵심 기능 + 감탄사 |
| 제품2 | 9~18초 | "두 번째," + 핵심 기능 + 감탄사 |
| 제품3 | 18~25초 | "세 번째," + 핵심 기능 + CTA |
| 클로징 | 25~28초 | 마무리 멘트 + "제품정보는 프로필 링크에!" |

### 유형 B — 1종 집중 (40~60초)

`여행갈때 필수템 (제품 1개).md` 구조 참고.
제품 기능을 하나씩 순서대로 보여주며 자연스럽게 연결.

## 화면 구성 작성 규칙

- 💚 컷별 **초 단위**로 상세 지정 (필수)
- 컷당 **1~2초**, 빠른 전환
- 순서: 제품 등장 → 핵심 기능 시연 → 결과/효과

```
💚 화면 구성:
- 벽에 선반 부착하는 장면 (1초)
- 세제 여러 개 올리는 장면 (1.5초)
- 물 뿌리고 물빠짐 클로즈업 (2초)
- 정리된 욕실 전경 (1.5초)
```

## 실행 단계

### Step 1: 예시 파일 숙지

`.claude/skills/script-writer/대본작성/예시/` 폴더 예시 4개를 읽고
문장 길이, 연결 방식, 화면 구성 패턴을 파악한다.

### Step 2: 유형 확인

01_products.md에서 유형 A(3종 묶음) / B(1종 집중) 확인.

### Step 3: 대본 작성

지침과 예시 패턴을 적용하여 대본 작성.
**작성 중 제품명이 들어가면 즉시 기능/특징 표현으로 교체.**

### Step 4: 자가 검수

- [ ] 오프닝이 "삶의 질 확 올라가는..."으로 시작하는가 (B형 예외)
- [ ] 대본 어디에도 제품명/브랜드명이 없는가
- [ ] 한 제품 소개가 10~12초 이내인가
- [ ] 연결어미로 끊기지 않고 이어지는가
- [ ] 화면 구성이 컷별 초 단위로 작성됐는가
- [ ] 클로징 CTA가 포함됐는가
- [ ] **인접 컷에서 동일/유사 정보가 중복되지 않는가 (DRY)**
  - 같은 명사·동사가 두 컷에 모두 등장하면 의미 중복 의심
  - 한 컷의 정보가 다음 컷의 부분집합/상위집합인 경우 통합 검토
  - 모든 컷을 순회하며 "이 컷이 없어도 메시지가 전달되는가" 자문

## 완료 후 액션 (절대 준수)

### 🚨 대본 작성/수정 시 필수 confirm 절차

대본은 영상의 핵심 자산이며 TTS 생성 시 비용이 발생하므로,
**최초 작성한 직후 OR 수정한 직후에는 반드시 다음 흐름을 따른다:**

1. 03_script.md 저장
2. 사용자에게 변경 사항을 명확히 보여줌 (전체 대본 또는 diff 형태)
3. **명시적 confirm 요청**: "수정할 부분 있으신가요? 없으면 '다음 단계'라고 해주세요"
4. 사용자가 **"다음 단계" / "확정" / "OK" / "진행"** 등을 명시적으로 답하기 전까지
   - TTS 재생성, sources 매핑, 다음 단계 진행 등 **그 어떤 후속 작업도 절대 시작하지 않는다.**

### ⚠️ Auto 모드 예외 없음

Claude Code Auto 모드가 활성화되어 있어도 이 룰은 **예외 없이 적용**된다.
이유: TTS 재생성은 비용·시간이 들고, 대본은 사용자 검토가 가장 중요한 단계.

### 수정 요청 처리

수정 요청을 받으면:
1. 해당 구간만 정확히 수정 후 파일 업데이트
2. 변경 사항을 다시 보여주고 confirm 재요청 (위 1~4 반복)
3. 사용자가 명시적으로 "다음 단계"라고 답할 때까지 반복

### 절대 금지 사항
- ❌ 대본 수정 후 자체 판단으로 TTS 재생성
- ❌ "수정 사항 반영 완료. 다음 단계로 진행합니다" 같은 자가 진행 멘트
- ❌ "대본 수정 후 자동으로 Step 3 실행" 류의 패턴
